# Дизайн миграции с Marzban на 3X-UI

## Цель

Перевести выдачу VPN, Mini App и админские операции с Marzban API на 3X-UI API.

## Базовые решения

- Панель: `3X-UI`
- Протокол: `VLESS + TCP + Reality`
- Формат выдачи: `subscription`
- Имя подписки: `LabGuard`
- Имя сервера внутри подписки: `Финляндия`
- Сейчас используется один inbound, но архитектура не должна мешать добавить несколько стран и серверов позже.

## Архитектура

- Вводится новый клиент панели `XUIClient` вместо `MarzbanClient`.
- Публичные сценарии бота и Mini App продолжают работать через единый объект панели, но реализация теперь идет через 3X-UI.
- Поле `subscription_url` сохраняется как раньше.
- Для идентификатора клиента панели добавляется нейтральное поле `panel_client_id` в БД. Старое `marzban_id` не используется новой логикой.

## Конфиг

Новые переменные окружения:

- `XUI_BASE_URL`
- `XUI_USERNAME`
- `XUI_PASSWORD`
- `XUI_INBOUND_ID`
- `XUI_SUBSCRIPTION_NAME=LabGuard`
- `XUI_SERVER_NAME=Финляндия`
- `XUI_VERIFY_TLS`

Старые `MARZBAN_*` после миграции должны быть удалены из runtime-логики.

## Поведение

### Выдача VPN

- Пользователь получает trial как и раньше.
- Бот создает клиента в указанном inbound 3X-UI.
- Клиент получает уникальный `email`, построенный из Telegram ID/username.
- Бот получает subscription link и нормализует имена:
  - подписка: `LabGuard`
  - сервер: `Финляндия`

### Статус и Mini App

- `/app/api/status` и `/app/api/get-vpn` читают пользователя через 3X-UI.
- Если в БД уже есть сохраненная подписка, она остается источником истины для ссылки, но при необходимости обновляется через панель.
- Трафик и online status читаются из 3X-UI.

### Admin операции

- deactivate -> disable client in 3X-UI
- delete -> remove client in 3X-UI
- update trial -> update expiry / create if absent

## Расширяемость

- Пока конфиг указывает на один inbound.
- В дальнейшем можно заменить один `XUI_INBOUND_ID` на таблицу серверов и маппинг стран без полной переделки панели-клиента.

## Этапы

1. Добавить `XUIClient` и новые env.
2. Перевести выдачу и статус Mini App.
3. Перевести админские операции.
4. Перевести метрики.
5. Выпилить Marzban-зависимости.
